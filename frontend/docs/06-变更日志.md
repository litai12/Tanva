# Changelog

All notable changes to this project are documented here.

This file follows the Keep a Changelog style, and the project aims to use Semantic Versioning.

## [Unreleased]

### Added
- Workspace：AI 设置新增“图片输入方式”（画布优先 / 节点优先）。选择节点优先后，外部图片粘贴、拖拽到画布和图片上传按钮会自动创建 Flow Image 节点；画布优先保持原有落画布行为。
- Admin/Auth：登录提醒弹窗改为活动弹窗结构，支持上半部分图片/视频媒体、下半部分受限富文本，以及底部主/次按钮。管理员可在 `/admin` → 系统设置 → 登录提醒维护 `login_notice` 的开关、媒体、文字、字体、字号和颜色；用户登录后通过 `GET /api/settings/login-notice` 读取配置，开启时需手动关闭弹窗。
- 前端使用说明文档重写（docs/02-使用说明.md）
- Flow：画板右键菜单新增“节点打组 (G)”，复用原 `G` 快捷键的选中节点打组逻辑。
- Flow：新增点击式连接交互，单击输出句柄后连线跟随鼠标，再单击目标输入句柄完成连接；右键菜单新增“批量连接输出”，可将选中节点输出端口批量连接到目标节点，并在句柄未渲染时按节点类型兜底生成跟随线；待连接期间锁定画板滚轮/拖拽视口，批量连接支持多 Prompt 输出接入可共享文本输入。
- Flow：按住 `Shift` 点击连线可直接删除该连线，并复用现有删除链路同步清理标签编辑态与历史记录。
- Flow：连线透明命中热区从默认值扩大到 32px，提升点击选中和 `Shift` 点击删除的命中率；按住 `Shift` 悬停连线时会显示红色减号删除光标，不改变连线视觉粗细。
- Global History：全局历史新增图片/视频统一媒体展示能力，支持识别视频记录、显示视频封面/播放入口、详情弹窗内播放视频，并在 AI Chat Seedance 视频生成成功后写入全局历史。
- Workspace：右上角顶栏新增 Nano Banana/Gemini/GPT-Image-2 生文/生图路线快捷切换，可直接在普通路线与尊享路线间切换，并复用设置面板中的全局 `bananaImageRoute` 状态；下拉中显示今日普通/尊享路线成功率。
- Workspace：设置 → 高级新增“帧检测”开关，FPS overlay 不再开发环境默认自动打开，并新增 `Canvas` 模式覆盖画布平移/移动帧检测。
- Canvas：绘制工具新增箭头模式，工具栏可选择箭头并支持点击-点击或拖拽绘制；图层面板会识别箭头图元并显示箭头类型。
- Backend：新增 `GET /api/ai/banana-route-success-rates`，按客户端时区汇总当天 Banana normal/stable 路线成功率，供工作区顶部路线切换使用。
- Project Manager：项目管理弹窗卡片改为懒加载当前页项目内容并展示多图预览宫格，每页展示 12 个项目，重命名/删除改为悬浮 icon 操作。
- 新增专题文档：
  - 认证与项目系统（docs/17-认证与项目系统.md）
  - 自动保存与版本管理（docs/18-自动保存与版本管理.md）
  - OSS 上传与资源管理（docs/19-OSS上传与资源管理.md）
  - 提示词优化功能说明（docs/20-提示词优化功能说明.md）
  - 图像历史与缓存系统（docs/21-图像历史与缓存系统.md）

### Changed
- Admin/Auth：内置登录活动弹窗改为 Seedance2.0 与 2026 Tanvas AI 创作公开赛双页轮播；Seedance 页保留原有展示样式，仅新增右侧浮动箭头，视频播放结束后自动滑到公开赛页，公开赛页可通过左箭头返回。
- README 更新：增加后端启动、文档导航与能力清单
- Project Load：项目内容缓存升级为账号隔离的 IndexedDB stale-while-revalidate；打开近期项目可先用本地内容水合，再后台校验远端版本，校验期间暂停自动/手动保存并在保存成功后缓存云端同款 sanitized 内容。
- Project Switch：顶部项目下拉点击项目后先关闭菜单，再异步触发 `projectStore.open()`；项目切换路径在已提前清空 Paper 时跳过导入前重复 `paper.project.clear()`，并把 Paper runtime rebuild 输入与耗时写入项目加载调试日志，用于区分缓存/网络延迟与 Paper/Flow 主线程重建成本。
- Flow：超过 80 节点时，画布平移/缩放、画布对象移动和 Flow 节点拖动期间会临时隐藏 MiniMap，交互停止后自动恢复，避免大量节点场景的概览 SVG 高频重绘；未超过阈值或交互空闲时 MiniMap 常驻显示。
- Flow：大量节点场景默认关闭 `onlyRenderVisibleElements`，避免节点进入视口时批量重挂载；低缩放硬降级暂不自动触发，80+ 节点的大图平移/缩放/节点拖拽期间启用 CSS 软降级，保留真实节点、文字、输入、按钮、resize 与连线，只隐藏可见的连接句柄圆点；同时新增视口邻近图片预热队列，按视口距离有限并发触发远程图片下载/解码，减少节点进入画面时的 decode/纹理上传抖动。
- Canvas：图片拖动改为移动中使用 DOM 层 `translate3d` 预览，鼠标松开后一次性提交 `imageInstances`；选中图片覆盖层同步走预览事件并暂停拖动中的 Paper bounds 轮询，减少多图拖动时的 React 高频更新。
- Flow：`Seedream` 节点新增豆包通道专属模型切换（`4.5/5.0`）；观猹通道固定使用 `5.0` 且不显示模型下拉。运行请求会携带 `modelVersion` 并映射到对应豆包 Model ID（`doubao-seedream-4-5-251128` / `doubao-seedream-5-0-260128`）。
- Flow：橡皮工具删除 Flow 连线恢复为点击命中模式，不再通过拖拽划线删除连线，避免 Paper.js 红色虚线轨迹残留。
- Flow：`Analysis` 节点支持多图输入分析；前后端 `/api/ai/analyze-image` 新增 `sourceImages`（兼容原 `sourceImage`），`gemini/gemini-pro/banana` 按多文件一并分析，`midjourney describe` 维持单图并在多图时返回明确错误。
- Flow：`Analysis` 节点 `text` 句柄支持多条 Prompt 输入，不再新线覆盖旧线；运行时会按连接顺序拼接为附加提示词。
- Flow：Image Split 分割改为运行时 `canvas/flow-asset`（Split 时不再强制上传 OSS）；保存链路会自动补传并将 `inputImageUrl` 替换为远程 URL/OSS key；Image Grid 支持基于 `splitRects` 裁切拼合并输出上传后的 OSS key/URL。
- Canvas：图片操作菜单新增“提取调色板”；点击后会基于当前图片提取主色，并在原图右侧生成一张独立的竖向调色板图片（`frontend/src/components/canvas/ImageContainer.tsx`）。
- Membership：标准版套餐卡片不再显示王冠图标，权益列表不再展示“基础月卡积分：500”首行。

### Fixed

- Flow：Prompt / Prompt Pro 节点文本区与 Generate 节点预设提示词在未选中时只作为预览展示，不再抢焦点或拦截画布/节点拖拽；选中节点后才恢复文本编辑、滚轮和 @ 图片插入交互。
- Canvas/Flow：Flow 覆盖层内滚轮缩放/平移改为 RAF 合并提交画布视口；`GridRenderer` 初始化兜底不再随每次缩放 tick 重跑，避免绕过已有的缩放重绘防抖。
- Admin/Auth：登录提醒次按钮可通过 `/__action__/wechat` 打开并固定工作区右上角微信二维码浮层；“开始创作”配置为 `/app` 且用户已在画布页时只关闭弹窗，不再刷新画布。
- Flow：Generate / Multi Generate / Generate Refer 运行态按钮不再把文案切换为 `Running...`，只保留灰色禁用态的 `Run`。
- Flow：Prompt Optimizer 预览输入框补齐中文输入法 composition 防护，拼音组词期间不再实时写回 Flow 节点数据，避免候选词确认前被重渲染打断并残留拼音片段。
- Flow：文字输入类 textarea 统一补齐中文输入法防护，覆盖 Prompt、Prompt Pro、Agent 提示词、Generate Refer、Video Analysis、MiniMax Music 和 Kling O3 自定义分镜等节点，候选词确认前不再被 Flow 节点数据写回打断。
- Membership：VIP 订阅页 UI 补齐迁移，顶部改为当前会员摘要栏，套餐卡片收紧高度与留白，并按档位强化卡片/按钮色彩区分；不改变订单创建和支付轮询逻辑。
- Library：库面板的全局历史/项目库补齐图片/视频统一媒体体验，视频记录支持封面、播放入口、详情播放、按真实媒体 URL 下载，并可作为视频资产发送/拖拽到画板，避免误走图片上传链路。
- AI Chat：项目内对话会话恢复边界对齐 `lt-dev9`，`ContextManager` 不再构造期自动读取全局本地会话；有项目 ID 时只从 `Project.content.aiChatSessions` 水合，避免旧本地历史串入当前项目对话框。
- AI Chat：缓存图上的“改文字/改成/替换文字”等编辑意图在工具选择失败时会兜底到 `editImage`，尊享路线下 Banana 工具选择会把上下文传给腾讯文本路由，避免退成普通文字回复。
- Flow：视频分析节点定价对齐 `lt-dev9`，静态默认改为 `60` 积分（`0.60` 元），并恢复按路线与档位动态计费：普通路线 Fast/Pro/Ultra 为 `60/90/120`，尊享路线为 `80/120/160`。
- Workspace：生文/生图线路的尊享路线 UI 对齐 `lt-dev9`，使用黄色王冠与 amber 状态色，避免显示为绿色星标。
- Canvas：精确局部修改的 Shift 框选会先按支持的画幅列表把选区校准到最近标准比例，再把校准后的选区 bounds、裁剪像素尺寸与比例传入 AI Chat，并使用 `precise-edit`/`lockToBounds` 锁定原选区占位，不再沿用普通编辑的右侧偏移或矩阵避让；合成回贴时同时传回原图外层 bounds，避免 `_merged.png` 自然尺寸改变画布显示比例；生成中占位层改为轻量虚线框加百分比数字，不再铺满半透明灰底或显示进度条。
- Canvas：图片工具栏“改文字”的 OCR 识别、内部文字检测和确认改图请求固定使用 Banana 普通路线，避免全局切到尊享路线后文字识别/替换不可用。
- Canvas：图片裁切结果会把当前画布显示比例烘进新图片，使裁切后图片的自然像素比例与新画布 bounds 保持一致，避免后续原位编辑/替换时再次出现自然比例和显示比例不一致。
- Canvas：智能抠图和极速抠图发送结果到画布时会携带当前图片的画布 bounds，使抠图结果沿用原图显示比例，不再按输出 PNG 的天然像素尺寸重新放大。
- Canvas：高清放大恢复为在原图右侧创建占位框并发送到画布，不再直接触发本地下载。
- Canvas：图片扩图 UI 对齐 `lt-dev9`，扩图按钮改用独立 Expand 图标，扩图蒙版提示改为红色区域，并在扩图选择结束后释放操作锁。
- Canvas：图片选中浮动工具栏的固定入口/更多菜单排序对齐 `lt-dev9`，固定展示 `生成节点 / 裁切 / 极速抠图`，其余图片能力继续按使用频次轮换进入主工具栏。
- Canvas：缩放/平移同时变化时统一走 `setViewport` 原子写入并跳过同值更新；全局触控板/手势缩放改为 RAF 合并提交，减少高频外部 store 更新导致的 React `Maximum update depth exceeded` 风险。
- Canvas/Flow：补齐剩余性能迁移；全局触控板/手势缩放只在 RAF 中提交视口，项目 canvas 视角同步增加 160ms 防抖与同值跳过，Flow 低细节模式改为外部订阅 canvas zoom，Image/ViewAngle 节点尺寸观测合并到 RAF。
- Canvas：非激活图片覆盖层不再订阅高频 `zoom/pan`；网格低缩放时减少线条渲染并合并缩放重绘，降低大画布缩放时的 React 与 Paper.js 更新压力。
- Flow：`GenerationProgressBar` 支持按节点 `runKey` 保持同一次运行的模拟进度，避免节点重渲染后进度回跳；节点分组运行时可点击停止按钮，当前节点完成后不再继续运行后续节点。
- Flow：补齐生成/视频/音频节点的 `progressStartedAt` 运行态时间戳，`FlowOverlay` 会在运行开始补写并在非运行/复制/模板导出时清理，避免进度重渲染回跳且不进入设计 JSON。
- Flow：Image Split 预览复用共享图片加载缓存并合并尺寸观测更新；输出端口在保留 `imageN` 的同时兼容历史 `imgN` 句柄，避免导入/保存归一化后的连线失效。
- Flow：Image Split 下游输入解析统一使用共享句柄 helper，`Generate/Agent/ImageCompress/ImageGrid/ViewAngle` 可一致解析 `imageN/imgN` 裁切来源；`VideoToGif` Run 按钮新增积分提示，视频输入节点的原生播放控件不再触发画布拖拽。
- Flow：Text Prompt 节点 resize 改为拖拽期间只更新本地预览、结束时一次性提交 ReactFlow 节点尺寸与位置，并在 resize 时暂停 node internals 高频同步。
- Flow：节点面板新增名称/类型/描述搜索；Quick Connect 只展示当前节点面板可见且可用的节点类型，避免隐藏节点从快捷连接入口被创建。
- Flow：反转滚轮模式下，节点添加面板内滚轮优先滚动面板内容，不再缩放或平移背后的画布。
- Flow：节点面板和 Quick Connect 按 `lt-dev9` 选择性迁移规则隐藏 `generateRef` 与 `sora2Video`，避免暴露内部/暂缓入口。
- Flow：项目内容水合到 ReactFlow 时会比较快照签名，相同内容直接跳过，减少 store 与 ReactFlow 之间的重复 hydrate/write-back。
- Flow：Text Chat 节点运行时只发送节点输入/手动输入拼接出的 prompt，不再自动混入全局聊天上下文，避免节点输出受画布外对话历史污染。
- Flow：Text Chat 节点移除底部联网搜索开关与状态行，运行时禁用联网搜索，并移除动态 resize / node internals 同步，只保留输入、Skill 与回复结果；Image Chat 节点保留英文标题、Run 与 Skill，仅将图片输入/结果占位等提示文案按中文显示。
- Flow：`analysis` 节点的外层面板、默认节点配置、接口兼容输出与后台导入模板统一显示为 `Image Chat`，描述文案改为“图像对话与提示词提取”，与节点内部标题保持一致。
- Flow：Image Chat 节点新增 `Custom` Skill；选中后显示可编辑提示词输入框，并继续支持从 `text` 句柄追加提示词。
- Flow：Text Chat 节点新增 Skill 预设（自定义、拆分镜头、提示词优化、中英文转换），并会即时读取上游文本节点的最新 patch；非自定义 Skill 可直接用内置提示词运行，也可叠加连接输入。
- Flow：Image Chat 节点新增 `Analysis` / `Prompt` / `JSON` Skill 预设选择；运行时使用所选 Skill 的内置提示词，并继续支持从 `text` 句柄追加提示词输入。
- Flow：Storyboard Split 支持输入参考格式（如 `分镜1`、`#1`、`|**1**|`）来自定义拆分边界；输出端口改为按解析结果自动生成，并在重新拆分后清理过期 prompt 输出与连线。
- Flow：Generate / Agent / Generate4 的上游参考图预览数量按节点模型档位限制（Fast=3、Pro=11、Ultra=14），避免 UI 与实际模型接图能力不一致。
- Flow：Generate / Agent / Generate4 / GeneratePro4 的连接容量与运行时请求也统一按模型档位限制参考图数量（Fast=3、Pro=11、Ultra=14），避免 UI 能接但运行时仍只发送 6 张参考图。
- Flow：GeneratePro4 节点补齐本地 Fast/Pro/Ultra 模型切换，运行积分预览会按当前节点模型与已连接参考图数量计算。
- Flow：视频生成节点成功后会把远程视频 URL 写入 Global History，覆盖 Wan、HappyHorse、Sora2、Seedance/Kling/Vidu 系列与 Tencent Speech 视频输出，不额外改动保存链路。
- Flow：Video Analysis 默认提示词会跟随当前语言同步，并在请求中显式携带 Banana route/channel 参数，和文本路由计费预览保持一致。
- Flow：Generate4 读取 Image Split 输入时统一兼容 `imageN/imgN` 句柄，Seedream 节点优先使用缩略图展示首图，Video Analysis 按钮和中文文案与其他运行节点保持一致。
- AI Chat：文本对话接口会透传后端 `metadata` 到消息和会话上下文；上下文提示收口为直接回答当前输入，避免模型输出内部意图分析。
- Backend：`/api/ai/text-chat` 的非 Gemini provider 路径会透传 `webSearchResult` 与 `metadata`；积分请求参数保留显式 `channelHint`，视频转存缓存增加 TTL 与最大条数上限。
- Backend：普通 `Kling` 视频节点对应的供应商模型切换为 `kling-v2-1`，其余 `Kling 2.6 / Kling O3` 逻辑保持不变（`backend/src/ai/services/video-provider.service.ts`）。
- Flow：节点弹窗在存在后端节点配置时，改为按归一化后的 Flow 节点类型去重并优先使用后端数据，避免前端 fallback 与后端配置叠加后出现重复的 `Kling` 节点（`frontend/src/components/flow/FlowOverlay.tsx`）。
