### Changed
- README 更新：增加后端启动、文档导航与能力清单
- Flow：`Analysis` 节点支持多图输入分析；前后端 `/api/ai/analyze-image` 新增 `sourceImages`（兼容原 `sourceImage`），`gemini/gemini-pro/banana` 按多文件一并分析，`midjourney describe` 维持单图并在多图时返回明确错误。
- Flow：`Analysis` 节点 `text` 句柄支持多条 Prompt 输入，不再新线覆盖旧线；运行时会按连接顺序拼接为附加提示词。
- Flow：Image Split 分割改为运行时 `canvas/flow-asset`（Split 时不再强制上传 OSS）；保存链路会自动补传并将 `inputImageUrl` 替换为远程 URL/OSS key；Image Grid 支持基于 `splitRects` 裁切拼合并输出上传后的 OSS key/URL。
- Flow：FPS 调试浮层新增画布缩放采样；开启 FPS 后，滚轮/触控板/缩放按钮触发画布缩放时会显示 `Zoom FPS`、最大帧耗时和长帧数。
- Canvas/Flow：优化画布缩放性能；Paper 图片层与 ReactFlow 节点层改为共享同一个 RAF viewport 快照，Flow 低细节/FPS 监听改为订阅式阈值更新，网格重建延后到缩放停顿后执行；非激活图片覆盖层不再订阅高频 `zoom/pan`，减少缩放期间长帧并避免节点/图片错帧。
- Canvas/Flow：图片上画布与 Flow Image 节点改为优先生成并渲染远程 WebP 缩略图；原图 URL/OSS key 继续保留给下载、高清预览、裁剪、编辑和 AI 输入等高清场景。
- Canvas：图片操作菜单新增“提取调色板”；点击后会基于当前图片提取主色，并在原图右侧生成一张独立的竖向调色板图片（`frontend/src/components/canvas/ImageContainer.tsx`）。

### Fixed
- Backend：普通 `Kling` 视频节点对应的供应商模型切换为 `kling-v2-1`，其余 `Kling 2.6 / Kling O3` 逻辑保持不变（`backend/src/ai/services/video-provider.service.ts`）。
- Flow：节点弹窗在存在后端节点配置时，改为按归一化后的 Flow 节点类型去重并优先使用后端数据，避免前端 fallback 与后端配置叠加后出现重复的 `Kling` 节点（`frontend/src/components/flow/FlowOverlay.tsx`）。
- Flow：隐藏 `Kling 2.6` 节点入口，不再在节点弹窗与快速连接菜单中显示；前端默认节点配置也同步移除该项（`frontend/src/components/flow/FlowOverlay.tsx`、`frontend/src/services/nodeConfigService.ts`）。
- Flow：修复节点弹窗中 `Kling` 系列条目因别名未映射导致点击后不创建节点的问题，现支持 `Kling / Kling 2.6 / Kling O1 / Kling O3` 等配置名映射到实际节点类型（`frontend/src/components/flow/FlowOverlay.tsx`）。
- Flow：修复 `Seedream` 节点左侧 `image` 输入句柄无法连接其他图片输出句柄的问题（`frontend/src/components/flow/FlowOverlay.tsx`）。
- Flow：修复 `Seedream` 节点中文文案乱码（帮助说明、按钮提示、等待态与尺寸下拉标签），并将尺寸标签分隔统一为 ASCII 文本，避免异常字符显示（`frontend/src/components/flow/nodes/Seedream5Node.tsx`）。
- Flow：优化 `Kling O3` 节点错误提示：将上游英文报错细分映射为简体中文，明确区分“图片像素尺寸不符合要求”与“图片格式不支持”；像素提示补充具体范围（最小边 `>=300px`、宽高比 `1:2.5~2.5:1`），格式提示补充支持类型（`JPG/JPEG/PNG`、单张 `<=10MB`），并保留原始错误便于排查（`frontend/src/components/flow/FlowOverlay.tsx`）。
- Flow：`Sora2` 节点切换为与 `Nano2` 同供应商链路（APIMart），后端统一复用 `NANO2_API_KEY`；节点模型收敛为 `sora-2` 与 `sora-2-pro`，移除 `sora-2-vip` 选项（`backend/src/ai/services/sora2-video.service.ts`、`backend/src/ai/dto/video-generation.dto.ts`、`backend/src/ai/ai.controller.ts`、`frontend/src/components/flow/nodes/Sora2VideoNode.tsx`、`frontend/src/components/flow/FlowOverlay.tsx`、`frontend/src/stores/aiChatStore.ts`、`frontend/src/services/aiBackendAPI.ts`）。
- Flow：`Sora2` 节点模式调整为两档：`Sora 2`（视频生成）与 `Sora 2 创建角色`（角色管理）；节点句柄、参数面板与请求分支会随模式切换。`Sora 2` 模式保留 `character` 视频输入句柄并可连接视频来源，`Sora 2 创建角色` 模式仅保留 `video` 输入并输出 `character` 句柄（`frontend/src/components/flow/nodes/Sora2VideoNode.tsx`、`frontend/src/components/flow/FlowOverlay.tsx`）。
- Flow：新增 `Sora2` 节点类型别名兼容，后台节点配置使用 `sora2` / `sora-2` / `sora2pro` / `sora-2-pro` 等 key 时，也能正确创建为 `sora2Video` 节点，修复节点面板点击无反应问题（`frontend/src/components/flow/FlowOverlay.tsx`）。
- Flow：取消 `Sora2` 节点在面板层的强制 `coming_soon` 覆盖，节点可见性与可用状态改为完全跟随后台节点配置（`frontend/src/components/flow/FlowOverlay.tsx`）。
- Flow：视频节点参考图按连线解析，支持 Image Split 切片作为输入。
- Flow: `Sora2` node now uses a `Mode` segmented selector (`Standard`/`Pro`), hides fixed defaults (`thumbnail=true`, `watermark=false`, `private=false`), defaults `Size=16:9` and `Duration=10s`, and only shows `Character Timestamps` when the `character` handle is connected to a video source.
- Flow：Image Split 生成的 Image 节点（`crop` 预览）在下游运行时会按裁切结果传参，避免仍使用完整原图。
- Flow：Image Split 配置恢复为“输出端口数量(1-50)”语义；网格切分按端口数自动推导（例如 2048x2048 要 512x512 切片可设 `16`）。
- Flow：Image Split（Worker 网格切分）不再对切片做“去白边裁切”，并保证输出数量严格等于端口数，避免切片尺寸被裁小/数量漂移。
- Flow：图像分析节点支持解析 Image Split / Image(crop) 的裁切输入，实际分析的是切片图（而非整图），并保持切片分辨率与尺寸正确。
- Flow：修复 Image Split 切片在下游裁切/拼合时分辨率被压缩（例如 base 图像只加载到缩略图时 1024 误变 200），并降低边缘白边概率。
- 前端图片转码：新增全局并发限流（暂定 10），收口图片生成/转化（`canvas.toDataURL/toBlob`、`FileReader.readAsDataURL`、`Response.blob`、`createImageBitmap/WebCodecs` 等）并在 AI Chat/Flow/画布等链路复用，降低多图场景瞬时内存峰值与卡顿风险。

## [v0.1.0] - 2025-09-16

### Added
- Center-based smart placement for new images:
  - `generate`: place new image to the right by 522px from cached center.
  - `edit`: place new image below by 522px from cached center.
- Cached image debug panel (`CachedImageDebug`): shows cached image ID, prompt, preview, center `(cx, cy)`, and latest mode (`generate|edit|blend|analyze|chat`).
- Event-driven updates:
  - `cachedImageChanged` fired when cache updates or clears.
  - `contextModeChanged` fired when recording an operation (mode change).
- Context cache extended with `latestBounds` and `latestLayerId`.
- When an image is placed on canvas or selected, cache is updated with its latest bounds.

### Changed
- Quick upload flow now accepts `smartPosition` and prioritizes it for precise placement.

### Notes
- If no cached position exists, the first image falls back to the default center `(0, 0)`.
- Layer info is optional for placement; calculations use the center point from bounds.
