# Liblib 风格画布导演台 1:1 改版

## 背景

Tanva 当前导演台由 Flow `directorConsole` 节点打开全屏 `DirectorConsoleModal`，内部已经具备角色、机位、3D 视口、属性面板、多镜头时间线、截图与灰模样片输出等能力，但产品外壳与 Liblib 当前导演工作区存在明显差异。

2026-07-18 使用 BrowserSkill 复用用户已登录的真实浏览器完成目标页取证。目标工作区的结构是：

- 顶部导航：项目名、当前工作区类型、`工作流 / 故事板`、同步/协作、账户与 Agent。
- 左侧悬浮工具列：添加节点、工具箱、素材库、角色库、历史记录、快捷键、教程、资产管理。
- 添加节点面板：文本、图片、视频、视频合成、导演台、音频、脚本、素材库，以及上传和生成历史。
- 中央：浅色无限画布，导演台是画布节点能力，而不是脱离工作流的独立产品壳。
- 底部视图控制：小地图、节点连线、网格吸附、缩放。

## 目标

以 LibTV 当前线上导演台为唯一产品规格，使画布导演台的布局、层级、视觉密度、入口、数据行为和输入/输出效果逐项对应。Tanva 原导演台多出的功能不再作为兼容目标，应从 UI 与执行路径移除。

“1:1”验收不以单个弹窗截图为准，而以完整导演工作区为准：画布壳层、导航、工具列、节点添加、导演台节点、导演台编辑态、资产入口、故事板入口、视口控制均需覆盖。

## 约束

- 不复制 Liblib 商标、账号标识、促销内容或受保护素材；只复刻导演工作区的信息架构、通用交互和视觉布局。
- 保留 Tanva 的账户、权限、项目保存与远程资源安全基础设施，但导演台产品语义、控件和执行行为以 LibTV 为准。
- `Project.contentJson` 仍只持久化远程 URL/路径；运行时截图或缩略图必须在保存前上传并替换，禁止新增 `data:`/`blob:`/裸 base64 持久化路径。
- 允许废弃原 directorConsole 的灰模样片、镜头片段编排、群演广播、自定义动作库、飞行运镜等额外能力；旧数据只需安全读取/忽略，不要求旧功能继续可操作。
- 改动聚焦导演工作区；不顺带重构无关 Flow 节点。

## 设计方向

1. 先抽出导演工作区视觉 token 与 shell 组件，建立浅色画布、半透明浮层、圆角工具按钮、统一面板阴影和选中态。
2. 将导演台入口提升为添加节点面板的一等节点，并与文本/图片/视频/脚本/素材库形成目标顺序。
3. 对齐顶部 `工作流 / 故事板` 双视图，并确保切换不丢失当前 Flow 数据。
4. 将素材库、角色库、历史记录和资产管理组织为同一左侧工具列入口。
5. 重写 `DirectorConsoleModal` 与时间轴：摄像机只保留 LibTV 的属性/截图，角色只保留属性/姿势/运动轨迹；时间轴改为属性关键帧轨道。删除 Tanva 原有 POV、镜头预设、景深元数据、路径运镜、飞行录制、灰模视频和镜头片段 TimelinePanel。
6. 输入句柄采用 `target` 并将上游当前图片作为全景；输出采用 `source`，截图发送后创建远程图片节点并自动连线，节点内容、位置和视觉效果对齐 LibTV。
7. 建立可重复的浏览器视觉验收：固定视口、固定示例 scene、目标与 Tanva 对照截图、关键控件 smoke test。

## 素体模型硬验收

- `男性 / 女性 / 宽厚 / 健壮 / 纤细 / 少年 / 儿童 / 二头身` 必须是八套独立、许可清晰的模型/体型资产映射，禁止继续用同一个 X Bot 仅靠高度或 `widthScale` 冒充。
- 每套逐项验收网格轮廓、头身比、肩胯比例、四肢比例、默认材质、默认颜色、落地高度、初始朝向、包围盒和摄像机截图表现。
- 每套必须具备可用人形骨架；同一 LibTV 姿势预设需要经过骨骼重定向后在八套素体上保持语义一致，儿童与二头身不能直接使用成人骨架比例缩放代替。
- 添加角色菜单还需覆盖空对象、群众 3×3、高斯泼溅、本地上传与几何体；这些对象在场景树、变换、隐藏/锁定和截图中的行为均纳入验收。
- 无法证明许可允许再分发的 LibTV 二进制模型不得直接抓取或提交；必须使用许可兼容的等效模型，并把仍有可见差异的素体标记为未完成。

## 验收证据

- BrowserSkill 对目标 Liblib 工作区的最新 snapshot/截图或可复现取证记录。
- Tanva 本地导演工作区固定视口截图，覆盖默认态、添加节点面板、导演台编辑态、故事板态。
- 旧 directorConsole 节点数据加载、角色/机位编辑、时间线播放、截图发送到画布均通过。
- `npm run lint` 与 `npm run build` 通过；新增的交互测试通过。
- 设计 JSON 检查未引入新的 inline 图片持久化。
