# 技术设计: 前端统一 fetch 鉴权入口

## 技术方案
### 核心技术
- React + TypeScript
- 统一 `fetchWithAuth` / 封装统一入口（可扩展配置）

### 实现要点
- 建立单一入口（例如 `request.ts` / 扩展 `authFetch`）
- 所有现有 `fetch` 调用统一替换为该入口
- 对公共/第三方/OSS 请求提供配置项，允许携带 token 但不强制鉴权
- 保留 401/403 触发退出登录逻辑

## 安全与性能
- **安全:** 401/403 统一处理，降低鉴权遗漏风险
- **性能:** 不增加额外网络请求，仅集中封装

## 测试与部署
- **测试:** 抽查鉴权接口/公共接口/第三方接口均可正常请求；401 触发退出登录
- **部署:** 前端常规发布流程
