# 平台工作流类型和分成模式说明

## 一、工作流类型

### 1. 工作流概述

Tanva 平台采用**可视化节点工作流**模式，用户通过拖拽和连接不同类型的节点来构建复杂的AI创作流程。工作流支持多种节点类型，按功能分为五大类：

### 2. 节点分类体系

#### 2.1 文字类节点（Text Nodes）
- **功能**：提示词输入、文本处理与拆分
- **节点类型**：
  - `textPrompt` - 基础提示词节点
  - `textPromptPro` - 专业提示词节点（Beta）
  - `textChat` - 文本对话节点
  - `textNote` - 文本备注节点
  - `promptOptimize` - 提示词优化节点
  - `storyboardSplit` - 分镜脚本拆分节点

#### 2.2 图像类节点（Image Nodes）
- **功能**：图像输入、生成与编辑
- **节点类型**：
  - `image` - 基础图像节点
  - `imagePro` - 专业图像节点（Beta）
  - `camera` - 相机捕获节点
  - `generate` - 单图生成节点
  - `generate4` - 四图生成节点
  - `generatePro` - 专业生成节点（Beta）
  - `generatePro4` - 四图专业生成节点（Beta）
  - `generateRef` - 参考图生成节点
  - `midjourney` - Midjourney 生图节点
  - `nano2` - Nano2 生图节点
  - `analysis` - 图像分析节点
  - `imageGrid` - 图像网格节点
  - `imageSplit` - 图像分割节点

#### 2.3 视频类节点（Video Nodes）
- **功能**：视频输入、生成与分析
- **节点类型**：
  - `video` - 基础视频节点
  - `sora2Video` - Sora2 视频生成节点
  - `sora2Character` - Sora2 角色生成节点
  - `wan26` - Wan2.6 视频生成节点
  - `wan2R2V` - Wan2.6 参考视频节点
  - `klingVideo` - 可灵 Kling 视频节点
  - `kling26Video` - 可灵 Kling 2.6 视频节点
  - `klingO1Video` - 可灵 Kling O1 视频节点
  - `viduVideo` - Vidu 视频节点
  - `doubaoVideo` - 豆包 Seedance 视频节点
  - `videoAnalyze` - 视频分析节点
  - `videoFrameExtract` - 视频帧提取节点

#### 2.4 3D类节点（3D Nodes）
- **功能**：三维模型处理
- **节点类型**：
  - `three` - 3D模型节点

#### 2.5 其他节点（Other Nodes）
- **功能**：辅助能力节点
- 包括各种工具类和辅助处理节点

### 3. 工作流构建方式

用户可以通过以下方式构建工作流：

1. **线性工作流**：节点按顺序连接，数据单向流动
   ```
   Prompt Node → Generate Node → Image Node
   ```

2. **分支工作流**：一个节点输出连接到多个节点
   ```
   Prompt Node → Generate Node → Image Node 1
                      ↓
                   Image Node 2
   ```

3. **聚合工作流**：多个节点输出汇聚到一个节点
   ```
   Prompt Node 1 ─┐
   Prompt Node 2 ─┼→ Text Chat Node → Generate Node
   Prompt Node 3 ─┘
   ```

4. **复杂工作流**：包含多种节点类型的复合流程
   ```
   Image Node → Analysis Node → Prompt Optimize Node → Generate Node → Video Node
   ```

### 4. 工作流执行模式

- **本地执行**：部分节点（如 `textChat`, `promptOptimize`, `analysis`, `videoAnalyze`）支持本地运行
- **云端执行**：AI生成类节点需要调用云端API
- **混合执行**：工作流可以同时包含本地和云端节点

## 二、分成模式

### 1. 当前分成机制：邀请奖励系统

平台目前采用**邀请奖励机制**，而非传统意义上的消费分成模式。

#### 1.1 邀请奖励规则

- **奖励金额**：每成功邀请1人，邀请人获得 **1000积分** 奖励
- **奖励上限**：每个用户最多可获得 **5次** 邀请奖励（总计5000积分）
- **触发条件**：被邀请用户完成**首次图像生成**后，系统自动发放奖励给邀请人

#### 1.2 邀请流程

1. **生成邀请码**：用户自动获得专属邀请码（格式：`TANVAS-XXXX`）
2. **分享邀请**：通过邀请链接或邀请码分享给他人
3. **注册使用**：被邀请人使用邀请码注册并完成首图生成
4. **自动发奖**：系统检测到被邀请人完成首图生成后，自动发放奖励给邀请人

#### 1.3 奖励状态

- `pending` - 待核验：已使用邀请码注册，但尚未完成首图生成
- `rewarded` - 已发奖：已完成核验并发放奖励

### 2. 积分系统说明

#### 2.1 积分获取方式

- **新用户注册**：赠送 1000 积分
- **每日签到**：连续签到7天，每天50积分，满7天额外奖励150积分
- **邀请奖励**：成功邀请用户获得1000积分（最多5次）
- **充值购买**：通过支付系统购买积分

#### 2.2 积分消耗

不同服务消耗不同积分，例如：
- **图像生成**：20-120积分/次（根据模型和分辨率）
- **视频生成**：600-1600积分/次（根据模型）
- **图像编辑**：30积分/次
- **文字对话**：10积分/次

### 3. 与传统分成模式的区别

| 特性 | 当前邀请奖励模式 | 传统消费分成模式 |
|------|----------------|----------------|
| 奖励触发 | 被邀请人完成首图生成 | 被邀请人每次消费 |
| 奖励金额 | 固定1000积分/人 | 按消费金额比例分成 |
| 奖励上限 | 5次（5000积分） | 通常无上限 |
| 持续性 | 一次性奖励 | 持续分成 |
| 适用场景 | 拉新获客 | 长期合作 |

### 4. 未来可能的分成模式

虽然当前代码中未实现，但平台架构支持扩展以下分成模式：

1. **消费分成模式**：被邀请人每次消费，邀请人按比例获得分成
2. **多级分成模式**：支持多级邀请关系，每级按比例分成
3. **合作伙伴模式**：为机构/企业提供定制化分成方案
4. **API调用分成**：基于API调用量的分成模式

## 三、如何向他人说明

### 工作流类型说明要点

1. **可视化节点工作流**：通过拖拽节点、连接节点的方式构建AI创作流程
2. **五大节点类别**：文字、图像、视频、3D、其他辅助节点
3. **灵活组合**：支持线性、分支、聚合等多种工作流模式
4. **本地+云端**：部分节点本地执行，AI生成类节点云端执行

### 分成模式说明要点

1. **邀请奖励机制**：成功邀请用户可获得1000积分奖励
2. **奖励上限**：每人最多5次邀请奖励（5000积分）
3. **自动发放**：被邀请人完成首图生成后自动发放
4. **非消费分成**：当前是固定奖励，不是按消费比例分成
5. **积分用途**：可用于平台内各种AI服务消费

### 常见问题回答

**Q: 工作流类型是什么意思？**
A: 工作流类型指的是平台支持的节点分类，包括文字处理、图像生成、视频生成、3D处理等五大类，用户可以通过连接这些节点来构建复杂的AI创作流程。

**Q: 分成模式是怎么算的？**
A: 目前平台采用邀请奖励机制，不是传统意义上的消费分成。每成功邀请1人（被邀请人完成首图生成），邀请人获得1000积分，最多5次。这是固定的一次性奖励，不是按被邀请人的消费比例持续分成。

**Q: 能获得持续分成吗？**
A: 当前版本不支持持续分成，只有一次性邀请奖励。未来可能会推出基于消费的分成模式。

**Q: 邀请奖励有上限吗？**
A: 是的，每个用户最多可以获得5次邀请奖励，总计5000积分。
