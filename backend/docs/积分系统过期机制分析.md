# 积分系统过期机制分析

## 一、概述

积分系统确实分为两部分：
- **会消失的积分（临时积分）**：普通用户的每日签到积分，7天后过期
- **永久保留的积分（永久积分）**：充值、注册赠送、邀请奖励、管理员添加等所有其他类型的积分

## 二、积分类型分类

### 2.1 会过期的积分（临时积分）

#### 每日签到积分（DAILY_REWARD）

**过期规则**：
- **普通用户**：签到积分在7天后过期（`DAILY_REWARD_EXPIRE_DAYS = 7`）
- **付费用户**：签到积分永不过期（`expiresAt = null`）

**实现位置**：
```1273:1274:Tanva/backend/src/credits/credits.service.ts
    // 计算过期时间：付费用户永不过期(null)，普通用户7天后过期
    const expiresAt = isPaid ? null : new Date(Date.now() + DAILY_REWARD_EXPIRE_DAYS * 24 * 60 * 60 * 1000);
```

**过期清理机制**：
- 定时任务：每天凌晨2点执行过期积分清理（`@Cron(CronExpression.EVERY_DAY_AT_2AM)`）
- 清理逻辑：查找所有 `expiresAt <= now` 且 `isExpired = false` 的签到积分记录
- 扣除方式：从用户余额中扣除过期积分，创建 `EXPIRE` 类型的交易记录

**关键代码**：
```1426:1547:Tanva/backend/src/credits/credits.service.ts
  async cleanupExpiredDailyRewards(): Promise<{ processedUsers: number; totalExpiredCredits: number }> {
    const now = new Date();

    // 查找所有已过期但未处理的签到积分记录
    const expiredTransactions = await this.prisma.creditTransaction.findMany({
      where: {
        type: TransactionType.DAILY_REWARD,
        expiresAt: { lte: now },
        isExpired: false,
        amount: { gt: 0 }, // 只处理正数（获得积分的记录）
      },
      include: {
        account: true,
      },
    });

    if (expiredTransactions.length === 0) {
      return { processedUsers: 0, totalExpiredCredits: 0 };
    }

    // 按用户分组处理
    const userTransactions = new Map<string, typeof expiredTransactions>();
    for (const tx of expiredTransactions) {
      const userId = tx.account.userId;
      if (!userTransactions.has(userId)) {
        userTransactions.set(userId, []);
      }
      userTransactions.get(userId)!.push(tx);
    }

    let processedUsers = 0;
    let totalExpiredCredits = 0;

    for (const [userId, transactions] of userTransactions) {
      // 再次确认不是付费用户（双重检查）
      const isPaid = await this.isPaidUser(userId);
      if (isPaid) {
        // 付费用户：将这些记录标记为永不过期
        await this.prisma.creditTransaction.updateMany({
          where: {
            id: { in: transactions.map(t => t.id) },
          },
          data: {
            expiresAt: null,
            isExpired: false,
          },
        });
        continue;
      }

      // 计算该用户需要清除的过期积分总额
      const expiredAmount = transactions.reduce((sum, t) => sum + t.amount, 0);

      if (expiredAmount <= 0) continue;

      // 获取用户当前余额
      const account = await this.prisma.creditAccount.findUnique({
        where: { userId },
      });

      if (!account) continue;

      // 实际扣除的积分不能超过当前余额
      const actualDeduct = Math.min(expiredAmount, account.balance);

      if (actualDeduct > 0) {
        await this.prisma.$transaction(async (tx) => {
          // 扣除过期积分
          const newBalance = account.balance - actualDeduct;
          await tx.creditAccount.update({
            where: { id: account.id },
            data: { balance: newBalance },
          });

          // 创建过期扣除记录
          await tx.creditTransaction.create({
            data: {
              accountId: account.id,
              type: TransactionType.EXPIRE,
              amount: -actualDeduct,
              balanceBefore: account.balance,
              balanceAfter: newBalance,
              description: `签到积分过期清除（${transactions.length}笔）`,
              metadata: {
                expiredTransactionIds: transactions.map(t => t.id),
                originalExpiredAmount: expiredAmount,
              },
            },
          });

          // 标记原始交易记录为已过期
          await tx.creditTransaction.updateMany({
            where: {
              id: { in: transactions.map(t => t.id) },
            },
            data: {
              isExpired: true,
              expiredAmount: actualDeduct,
            },
          });
        });

        totalExpiredCredits += actualDeduct;
      } else {
        // 余额为0，只标记为已过期
        await this.prisma.creditTransaction.updateMany({
          where: {
            id: { in: transactions.map(t => t.id) },
          },
          data: {
            isExpired: true,
            expiredAmount: 0,
          },
        });
      }

      processedUsers++;
    }

    this.logger.log(`签到积分过期清理完成: 处理 ${processedUsers} 个用户, 清除 ${totalExpiredCredits} 积分`);
    return { processedUsers, totalExpiredCredits };
  }
```

## 四、积分交易类型（TransactionType）

```4:11:Tanva/backend/src/credits/dto/credits.dto.ts
export enum TransactionType {
  EARN = 'earn',
  SPEND = 'spend',
  REFUND = 'refund',
  ADMIN_ADJUST = 'admin_adjust',
  DAILY_REWARD = 'daily_reward',
```

**各类型说明**：
- `EARN`：获得积分（充值、注册赠送等）- **永不过期**
- `SPEND`：消费积分（API调用扣费）
- `REFUND`：退还积分（API调用失败退款）- **永不过期**
- `ADMIN_ADJUST`：管理员调整积分 - **永不过期**
- `DAILY_REWARD`：每日签到积分 - **普通用户7天后过期，付费用户永不过期**
- `EXPIRE`：过期清除记录（签到积分过期时创建）

## 五、总结

### 5.1 积分分类表

| 积分类型 | 交易类型 | 是否过期 | 过期时间 | 说明 |
|---------|---------|---------|---------|------|
| 每日签到积分（普通用户） | `DAILY_REWARD` | ✅ 是 | 7天 | 普通用户签到积分7天后过期 |
| 每日签到积分（付费用户） | `DAILY_REWARD` | ❌ 否 | 永不过期 | 付费用户签到积分永不过期 |
| 充值积分 | `EARN` | ❌ 否 | 永不过期 | 用户充值获得的积分 |
| 新用户注册赠送 | `EARN` | ❌ 否 | 永不过期 | 新用户注册时赠送的积分 |
| 邀请奖励 | `REFERRAL_REWARD` | ❌ 否 | 永不过期 | 邀请好友获得的奖励积分 |
| 管理员添加 | `ADMIN_ADJUST` | ❌ 否 | 永不过期 | 管理员手动添加的积分 |
| 退款积分 | `REFUND` | ❌ 否 | 永不过期 | API调用失败退还的积分 |

### 5.2 关键设计点

1. **只有签到积分会过期**：只有 `DAILY_REWARD` 类型的积分会根据用户类型设置过期时间
2. **付费用户特权**：付费用户的签到积分永不过期，这是付费用户的特权之一
3. **过期清理机制**：每天凌晨2点自动清理过期的签到积分
4. **双重检查**：清理任务会再次检查用户是否为付费用户，如果是则标记为永不过期
5. **余额保护**：过期清理时，实际扣除的积分不能超过用户当前余额

### 5.3 业务逻辑

- **普通用户**：签到积分7天后过期，鼓励用户持续使用或充值
- **付费用户**：签到积分永不过期，作为付费用户的额外福利
- **其他积分**：充值、注册、邀请等获得的积分都是永久积分，不会过期
