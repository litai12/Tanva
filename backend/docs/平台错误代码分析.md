## 🔍 错误代码分类

### 1. HTTP 状态码错误

#### 格式
```
HTTP_XXX
```
其中 `XXX` 为 HTTP 状态码（如 401, 403, 404, 500, 502, 503 等）

#### 定义位置
- **后端**: `Tanva/backend/src/ai/ai.controller.ts`
- **前端**: `Tanva/frontend/src/services/aiBackendAPI.ts`

#### 产生原因
当后端调用第三方 API 或内部服务时，收到非 2xx 的 HTTP 响应状态码。

#### 代码示例
```2167:2167:Tanva/backend/src/ai/ai.controller.ts
          return { success: false, error: { code: `HTTP_${response.status}`, message: data?.message || `DashScope HTTP ${response.status}`, details: data } };
```

```2243:2243:Tanva/backend/src/ai/ai.controller.ts
              code: `HTTP_${response.status}`,
```

#### 常见状态码
- **HTTP_401**: 未授权，API Key 无效或过期
- **HTTP_403**: 禁止访问，权限不足
- **HTTP_404**: 资源不存在
- **HTTP_429**: 请求频率过高（限流）
- **HTTP_500**: 服务器内部错误
- **HTTP_502**: 网关错误
- **HTTP_503**: 服务不可用

#### 处理逻辑
```139:145:Tanva/frontend/src/services/aiBackendAPI.ts
const parseHttpStatusFromErrorCode = (code?: string): number | null => {
  if (!code) return null;
  const match = code.match(/^HTTP_(\d{3})$/);
  if (!match) return null;
  const status = Number(match[1]);
  return Number.isFinite(status) ? status : null;
};
```

#### 可重试判断
```147:173:Tanva/frontend/src/services/aiBackendAPI.ts
const isRetryableImageGenerationError = (error?: {
  code?: string;
  message?: string;
}): boolean => {
  if (!error) return false;
  if (error.code === "NETWORK_ERROR") return true;

  const status = parseHttpStatusFromErrorCode(error.code);
  if (status !== null) {
    if (status === 408 || status === 429) return true;
    if (status >= 500) return true;
  }

  const message = String(error.message ?? "").toLowerCase();
  if (!message) return false;

  // 后端上游偶发返回空图（截图中对应：生成图像数据为空，无法上传。）
  const transientPatterns = [
    "生成图像数据为空",
    "no image data",
    "returned no image",
    "empty image",
  ];
  return transientPatterns.some((pattern) =>
    message.includes(pattern.toLowerCase())
  );
};
```

**可重试的 HTTP 错误码**:
- `HTTP_408` (请求超时)
- `HTTP_429` (限流)
- `HTTP_5XX` (服务器错误，500-599)

### 2. 网络错误 (NETWORK_ERROR)

#### 格式
```
NETWORK_ERROR
```

#### 定义位置
- **后端**: `Tanva/backend/src/ai/ai.controller.ts`
- **前端**: `Tanva/frontend/src/services/aiBackendAPI.ts`, `Tanva/frontend/src/services/aiImageService.ts`

#### 产生原因
1. **网络连接失败**: DNS 解析失败、连接超时、连接被拒绝
2. **请求异常**: fetch 抛出异常（非 HTTP 错误）
3. **网络中断**: 请求过程中网络断开

#### 代码示例
```2201:2201:Tanva/backend/src/ai/ai.controller.ts
        return { success: false, error: { code: 'NETWORK_ERROR', message: error?.message || String(error) } };
```

```451:451:Tanva/frontend/src/services/aiImageService.ts
          code: isTimeout ? "TIMEOUT_ERROR" : "NETWORK_ERROR",
```

#### 处理逻辑
- **可重试**: `NETWORK_ERROR` 通常可以重试
- **重试策略**: 最多重试 3 次，每次延迟 1 秒

```1624:1635:Tanva/backend/src/ai/services/sora2-video.service.ts
  private isRetryableVideoError(error: unknown): boolean {
    const code = (error as any)?.code as string | undefined;
    const message =
      error instanceof Error ? error.message : String(error ?? "");
    if (code?.startsWith("HTTP_5")) return true;
    if (code === "NETWORK_ERROR") return true;
    if (/load failed/i.test(message)) return true;
    if (/failed to fetch/i.test(message)) return true;
    if (/network.*error/i.test(message)) return true;
    if (/timeout/i.test(message)) return true;
    return false;
  }
```

### 5. Sora2 视频生成错误

#### 错误类型列表

| 错误代码 | 说明 | 产生原因 | 用户提示 |
|---------|------|---------|---------|
| **sora_content_violation** | 内容违反社区准则 | 提示词包含敏感内容（裸露、色情等） | "内容可能违反社区准则（涉及裸露、色情等）" |
| **sora_rate_limit** | API 调用频率过高 | 请求过于频繁，触发限流 | "请求过于频繁，请稍后重试" |
| **sora_invalid_prompt** | 提示词无效 | 提示词格式不正确或为空 | "提示词格式不正确" |
| **sora_timeout** | 生成超时 | 视频生成任务超时 | "生成超时，请重试" |
| **queued** | 任务在队列中 | 任务已提交，正在排队等待处理 | "任务正在排队，请稍后查看" |
| **processing** | 任务处理中 | 任务正在处理中 | "任务正在处理，请稍后查看" |

#### 定义位置
- **文档**: `Tanva/frontend/docs/31-Sora2API响应格式指南.md`

#### API 响应格式
```json
{
  "status": "failed",
  "error": {
    "type": "sora_content_violation",
    "message": "This content may violate our guardrails around nudity, sexuality, or erotic content."
  }
}
```

#### 处理逻辑
系统会解析 `error.type` 字段，并根据错误类型返回相应的用户提示。

#### 格式

#### 定义位置
- **后端**: `Tanva/backend/src/ai/providers/midjourney.provider.ts`

#### 成功码
- **code = 1**: 成功
- **code = 22**: 排队中（正常状态）

#### 错误处理
```152:158:Tanva/backend/src/ai/providers/midjourney.provider.ts

    if (data?.code && data.code !== 1 && data.code !== 22) {
      // code 1 = 成功, code 22 = 排队中
      const errorMsg = data.description || data.message || 'Unknown API error';
      this.logger.error(`[Midjourney] ${operation} API error: code=${data.code}, ${errorMsg}`);
      throw new Error(`Midjourney ${operation} failed: ${errorMsg}`);
    }
```

## 🔄 错误处理流程

### 前端错误处理流程

```
请求发起
  ↓
捕获异常
  ↓
判断错误类型
  ↓
├─ NETWORK_ERROR → 重试（最多3次）
├─ HTTP_408/429/5XX → 重试（最多3次）
├─ HTTP_401/403 → 回退到公开 API
├─ TIMEOUT_ERROR → 提示用户重试
└─ 其他错误 → 显示错误信息
```

### 后端错误处理流程

```
调用第三方 API
  ↓
检查 HTTP 状态码
  ↓
├─ 2xx → 检查业务错误码
│   ├─ 成功 → 返回数据
│   └─ 失败 → 抛出业务错误
├─ 4xx → 返回 HTTP_XXX 错误
├─ 5xx → 返回 HTTP_XXX 错误（可重试）
└─ 网络异常 → 返回 NETWORK_ERROR
```

## 🛠️ 错误代码使用建议

### 1. 统一错误格式
所有错误响应应遵循以下格式：
```typescript
{
  success: false,
  error: {
    code: string,        // 错误代码
    message: string,     // 错误消息
    details?: any,      // 错误详情（可选）
    timestamp?: Date     // 时间戳（可选）
  }
}
```

### 2. 错误代码命名规范
- HTTP 状态码错误: `HTTP_XXX`
- 网络错误: `NETWORK_ERROR`
- 超时错误: `TIMEOUT_ERROR`
- 业务错误: 使用描述性名称（如 `sora_content_violation`）

### 3. 可重试错误判断
```typescript
function isRetryableError(error: { code?: string }): boolean {
  if (error.code === "NETWORK_ERROR") return true;
  if (error.code === "TIMEOUT_ERROR") return true;
  
  const status = parseHttpStatus(error.code);
  if (status === 408 || status === 429) return true;
  if (status >= 500) return true;
  
  return false;
}
```

### 4. 用户友好的错误提示
- 技术错误代码不应直接展示给用户
- 应转换为用户可理解的中文提示
- 提供具体的解决方案建议

## 🔍 常见错误排查

### 1. NETWORK_ERROR
**可能原因**:
- DNS 解析失败
- 防火墙阻止
- 代理配置错误
- 服务器网络故障

**排查步骤**:
1. 检查服务器网络连接
2. 验证 DNS 配置
3. 检查防火墙规则
4. 查看代理设置

### 2. HTTP_401/403
**可能原因**:
- API Key 无效或过期
- 权限不足
- Token 过期

**排查步骤**:
1. 检查 API Key 配置
2. 验证 Token 有效性
3. 确认权限设置

### 3. HTTP_429
**可能原因**:
- 请求频率过高
- 触发限流策略

**排查步骤**:
1. 检查请求频率
2. 实现请求队列
3. 添加指数退避重试

### 4. sora_content_violation
**可能原因**:
- 提示词包含敏感词汇
- 内容违反社区准则

**解决方案**:
- 修改提示词，去除敏感内容
- 使用更中性的描述
