# Nano Banana 积分定价修改对比

## 修改说明

根据市场报价分析，对 Nano Banana 系列模型的积分定价进行了优化，实现了按分辨率差异化计费。

## 修改前后对比表

### 1. Nano Banana (Fast模式, gemini-2.5-flash-image-preview)

| 分辨率 | 修改前积分 | 修改后积分 | 修改前价格(元) | 修改后价格(元) | 变化 |
|--------|-----------|-----------|---------------|---------------|------|
| 1K     | 20        | **60**    | 0.20          | 0.60          | **+40积分** ⬆️ |

**说明**: Fast模式仅支持1K分辨率，定价从20积分调整为60积分。

## 技术实现变更

### 1. 配置文件修改 (`credits.config.ts`)

**修改前**:
```typescript
'gemini-3-pro-image': {
  serviceName: 'Nano banana Pro 生图',
  provider: 'gemini',
  creditsPerCall: 30,
  description: '使用 Nano banana Pro 模型生成高质量图像',
},
```

**修改后**:
```typescript
'gemini-3-pro-image': {
  serviceName: 'Nano banana Pro 生图',
  provider: 'gemini',
  creditsPerCall: 40, // 默认1K分辨率
  description: '使用 Nano banana Pro 模型生成高质量图像',
  resolutionPricing: {
    '1K': 40,
    '2K': 60,
    '4K': 120,
  },
},
```

### 2. 计费逻辑修改 (`credits.service.ts`)

**修改前**:
- 简单的4K判断，固定60积分
- 其他分辨率使用基础定价

**修改后**:
- 新增 `resolveImageResolutionCredits()` 方法
- 支持按分辨率（0.5K/1K/2K/4K）差异化计费
- 自动从 `resolutionPricing` 配置中读取对应积分

### 3. 新增方法

```typescript
private resolveImageResolutionCredits(
  serviceType: ServiceType,
  defaultCredits: number,
  requestParams: any,
): number
```

该方法会：
1. 检查是否为图像生成服务
2. 读取 `resolutionPricing` 配置
3. 根据 `requestParams.imageSize` 匹配对应积分
4. 如果未找到匹配，返回默认积分

## 定价策略分析

### 与市场对比

| 模型 | 分辨率 | Tanvas定价(元) | 官方价格(元) | 竞争对手平均(元) | 市场定位 |
|------|--------|---------------|-------------|----------------|---------|
| Nano Banana | 1K | 0.60 | 0.27 | 0.20-0.39 | 中等偏上 ✅ |
| Nano Banana-Pro | 1K | 0.40 | - | - | 新增定价 ✅ |
| Nano Banana-Pro | 2K | 0.60 | 0.924 | 0.15-1.04 | 中等偏上 ✅ |
| Nano Banana-Pro | 4K | 1.20 | 1.655 | 0.69-1.79 | 中等偏上 ✅ |
| Nano Banana-2 | 0.5K | 0.20 | 0.31 | 0.075-0.41 | 中等 ✅ |
| Nano Banana-2 | 1K | 0.30 | 0.46 | 0.069-0.62 | 中等 ✅ |

| Nano Banana-2 | 4K | 0.60 | 1.04 | 0.207-1.38 | 中等偏下 ✅ |

### 定价优势

1. **竞争力**: 所有分辨率定价均低于官方价格，高于最低价（147API），处于市场中等偏上水平
2. **合理性**: 按分辨率差异化定价，符合成本结构
3. **灵活性**: 可根据实际API成本调整 `resolutionPricing` 配置

## 测试建议

1. **功能测试**:
   - 测试各模型在不同分辨率下的积分扣费是否正确
   - 验证未指定分辨率时使用默认积分
   - 验证不支持的分辨率使用默认积分

2. **边界测试**:
   - 测试无效分辨率值
   - 测试空值/undefined情况
   - 测试大小写不敏感匹配

3. **集成测试**:
   - 测试前端节点调用后端API时的积分扣费
   - 验证交易记录中的描述信息包含分辨率
